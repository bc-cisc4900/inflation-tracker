# Project Description

## Overview
The Grocery Store Inflation Tracker is a Brooklyn College group project that tracks the prices of essential grocery items across multiple stores over time. The purpose of the project is to collect, standardize, store, and display local grocery price data so users can better understand weekly price changes, monthly averages, and inflation-related trends.

## Main Goal
The project is designed to compare grocery prices across several stores and observe how those prices change over time. The team collects price data weekly, calculates monthly averages, stores the data in a MariaDB database, and presents the information through a website.

## Stores
The project currently focuses on stores such as:
- Aldi
- Key Food
- Stop & Shop
- Trader Joe’s
- FreshDirect

Additional online vendors may be considered later if they improve the usefulness of the data.

## Data Collected
The project focuses on essential grocery items that represent realistic consumer purchases. Each item must be comparable across stores by keeping the following as consistent as possible:
- product type
- package size
- item quality
- unit price logic
- substitution rules

## Main Components
The project currently includes:
- weekly grocery data collection
- Google Sheets used for raw tracking and averages
- a MariaDB database for structured storage
- a website for presentation of the data
- a grocery importer / backend component
- project documentation and project board tracking

## Key Output
The most important output metric is the monthly average price for each tracked item. The team also wants the system to support a sliding 12-month view so the most recent 12 months of data are displayed in order.

## Current Development Status
The project has already established:
- a working GitHub repository
- a live website link
- a database design direction
- grocery item standardization rules
- weekly data collection workflow
- a project board for task management

## Project Philosophy
The project prioritizes functionality, consistency, and documentation. The team’s goal is to build a working, understandable system first, then improve automation, visualization, and refinement over time.
