Project Overview

This project grew from real data collection trips, spreadsheet standardization, database design, website development, filtering research, and ongoing testing. The website is meant to communicate that this is not just a spreadsheet embed—it is a full course project built around methodology, consistency, and long-term usability.

Weekly Collection

Prices are collected on a recurring weekly schedule so changes can be tracked consistently over time.

Monthly Averages

The main output is a monthly average built from weekly observations instead of a single snapshot.

Filtering Focus

The project is designed around exploring results by date, store, and item in a simple and understandable way.

Future Continuation

This project is meant to continue beyond one semester so future groups can build on the same structure.

How the System Works

The project is built as a pipeline. Grocery prices are collected and standardized, recorded in shared project materials, moved into the database, and then presented through the website in a format that is easier to read and eventually easier to filter.

1

Price Collection

Weekly grocery prices are collected from selected stores using consistent item definitions and substitution rules.

2

Google Sheets

Collected prices are organized into shared sheets, reviewed, averaged, and documented for consistency.

3

MariaDB Database

Structured records are stored in the database so the data can scale beyond static documents and support future queries.

4

Website Display

The website presents the project, explains the methodology, and serves as the public-facing layer for trend exploration.

Why Methodology Matters

Grocery inflation data can be affected by substitutions, product availability, sales, packaging changes, collection timing, and differences between stores. Because of that, this project emphasizes consistency, documentation, and structured averages so the results are more reliable and meaningful.

Consistency

Matching size, type, and brand as closely as possible helps keep results comparable from week to week.

Trend Analysis

The project is designed to highlight patterns over time rather than treat one recorded price as the final story.

Clear Navigation

Methods and Credits are separated into their own pages so the website stays organized and easier to understand.
